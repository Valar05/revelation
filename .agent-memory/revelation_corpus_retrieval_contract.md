# Revelation Corpus Retrieval Contract

## Purpose

Generated content should be recontextualized from corpus material, not invented from empty atmosphere. Every generated mission, follow-up, interlude, artifact, and base incident should name the source anchors it used.

## Required Anchor Mix

Every mission should use:

- at least one symbolic anchor from scripture, apocrypha, folklore, or occult/public-domain religious material,
- at least one procedural anchor from FEMA, CDC, OSHA, NIOSH, Army, or other approved official sources,
- at least one state/character anchor from Revelation project-authored docs or exemplars.

Follow-ups should use:

- the same root symbolic anchor or a clearly related anchor,
- one procedural or psychological/logistics anchor,
- a state key from the parent branch.

Interludes should use:

- one state key,
- one character or squad arc hook,
- one ordinary-life frame such as meal, smoke break, clinic, debrief, barracks, lab, logistics, or command corridor.


## Anchor Must Survive Into Play

A corpus anchor is not sufficient if it only supplies mood, theme, or a summary label. For every mission and major follow-up, the source must survive into at least three concrete places:

- `source_fingerprint`: a brief private note naming the source's usable circumstance, procedure, object, contradiction, or sequence.
- `structural_transfer`: how that source feature becomes a Revelation decision, hazard, evidence chain, or state change.
- `required_visible_details`: two or more player-facing objects, phrases, procedures, sounds, documents, timings, or bodily effects that must appear in prose or outcomes.
- `followup_payoff`: how the same anchor mutates later instead of becoming a repeated echo.

Weak anchor: "CDC isolation gives procedural tone."

Strong anchor: "Transmission-based precautions become form quarantine; wristbands and discharge packets are treated as carriers; the follow-up tests whether a clean bed/dirty form boundary can hold."

Weak anchor: "Genesis naming theme."

Strong anchor: "Calling and dividing become operational hazards: a meter prints names instead of numbers, clean/unclean vocabulary splits the team, and the next follow-up removes a name-shaped blank from the report."

## Corpus Influence Fields

Use this shape in generated records:

- `seed_id`
- `source_title`
- `source_chunk_id`
- `source_moment`
- `writing_influence`
- `room_application`
- `followup_application`
- `interlude_application`
- `license_category`
- `source_fingerprint`
- `structural_transfer`
- `required_visible_details`
- `followup_payoff`

## Forbidden Use

- Do not paste long source passages into player-facing prose.
- Do not use SCP content in the active corpus unless the project deliberately accepts CC BY-SA obligations.
- Do not turn public procedural sources into direct tactical instruction for players.
- Do not treat old anthropological/occult sources as factual modern science.
- Do not cite a corpus source that did not materially influence the event.

## Retrieval Priority

When multiple anchors are available:

1. Project-authored Revelation exemplars.
2. Active source corpus chunks.
3. Foundational/style/interlude docs.
4. User-provided exact premise.

## Output Requirement

Generated patches should include a short hidden/non-player-facing explanation of:

- why each corpus anchor was selected,
- which state keys are read and written,
- how follow-ups progress rather than echo,
- which interlude can carry the consequence.
